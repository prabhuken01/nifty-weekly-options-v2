================================================================================
THE CODE CHANGE
================================================================================

You asked: "what needs to be changed from postgresql to dbeaver"

Answer: NOTHING changes from PostgreSQL to DBeaver.

DBeaver IS the tool to configure PostgreSQL passwords.
Python script USES that same PostgreSQL password.

What changes: Only the DB_PASSWORD line in Python script.

Before:
  DB_PASSWORD = "postgres"  ← This is a placeholder

After:
  DB_PASSWORD = ""          ← If no password
  OR
  DB_PASSWORD = "postgres"  ← If password is "postgres"
  OR
  DB_PASSWORD = "your_password"  ← If password is something else

================================================================================
COMMON CONFUSION
================================================================================

Question: "Should I set password IN DBeaver or IN Python?"
Answer: Both! They must MATCH.

Question: "Does DBeaver have its own password?"
Answer: No. DBeaver stores the PostgreSQL password. You need to know it.

Question: "What if I don't know the PostgreSQL password?"
Answer: Use DBeaver to find it (by testing blank, postgres, etc.)
        Or reinstall PostgreSQL and set a new password.

Question: "Can I use different passwords for DBeaver and Python?"
Answer: NO! They must be identical, or script will fail with
        "password authentication failed"
